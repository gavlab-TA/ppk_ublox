# ppk_ublox

Two programs for cm-level **post-processed kinematic (PPK)** positioning from a
u-blox **ZED-F9P**, corrected against a **NOAA CORS** reference station — no live
corrections, no base of your own, no website service.

1. **`ubx_logger`** — configures the F9P for raw observables and streams them to a
   `.ubx` file in the field.
2. **`ppk_solve`** — takes that log plus a folder of CORS RINEX and produces a
   cm-level ECEF / LLA track (demo5 RTKLIB engine underneath).

All the code, build instructions, and usage live in **[`ublox-ppk/`](ublox-ppk/README.md)** —
start there.

```
ppk_ublox/
  ublox-ppk/   the two programs + full README   [tracked]
  rinex/       CORS base RINEX for a session     [gitignored]
  work/        pipeline inputs/outputs           [gitignored]
```

## Quick start

```bash
cd ublox-ppk
cmake -B build -S . && cmake --build build -j     # fetches + builds RTKLIB demo5

# log raw observables from an F9P over USB
./build/logger/ubx_logger --port /dev/ttyACM0 --out ../work/rover.ubx --rate 5

# solve against a folder of CORS RINEX (see ublox-ppk/README.md §2 to fetch it)
./build/ppk/ppk_solve \
  --rover ../work/rover.ubx \
  --base-dir ../rinex \
  --conf build/ppk/f9p_kinematic.conf \
  --base-ecef 421642.5824 -5362176.5353 3416663.3284 \
  --out ../work/ppk_positions.csv        # also writes raw_positions.csv
```

Output is plain CSV — one header row, then one row per epoch — with GPS/UTC time,
LLA, ECEF, per-epoch quality and 1σ. **Filter on `quality == "fix"`;** float is
not a product. Column meanings and everything else are documented in
`ublox-ppk/README.md`.

## What is tracked

Only source is committed (~200 KB): the `ublox-ppk/` programs and their README.
The data directories are large binary GNSS logs kept out of the repo — the base
RINEX is public and re-downloadable (`ublox-ppk/README.md` §2), and `work/` is
regenerated by running the pipeline. See `.gitignore`.
